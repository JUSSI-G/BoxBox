Architecture

Three-file pipeline

One command runs everything. No hardcoded correction factors — the RF drives the simulation.

01
analyser.py
Variance Analyser
Loads pitstops.csv + Kaggle CSV. Computes pit window errors and execution penalties using the Heilmeier et al. (2018) tyre degradation model. Fetches grid positions from Ergast API (cached to data/f1_cache/).
→ window_penalty_s
→ exec_penalty_s
→ avg_window_error_laps
→ era_code
02
xgb.py
XGBoost Model
Trains XGBoostRegressor on 1994–2011, tests on 2012–2024. Feature importances per era. Decreasing variance/grid ratio = thesis claim. Saves dataset and passes trained model directly to f1.py.
→ f1_rf_dataset.csv
→ feature importances
→ trained XGB model
→ f1_rf_results.png
03
f1.py
Championship Simulator
Receives trained model. Zeros variance features per driver-race, keeps grid + era fixed. Predicts points under both scenarios. Rebuilds standings. Flags seasons where the title would have changed.
→ f1_simulation_sweep.csv
→ title-flip seasons
→ f1_sweep_simulation.png
Research Question
THESIS CLAIM
"Does strategy variance predict race outcome, and does this relationship change across software eras — indicating that software reduced the impact of strategy errors over time?"

Heilmeier et al. 2020
Virtual Strategy Engineer. Applied Sciences. DOI: 10.3390/app10217805. Justifies quadratic optimisation for optimal pit strategy without opponents.
scikit-learn / XGBoost
Permutation Importance with Multicollinear Features. Justifies removal of total_variance_s from FEATURE_COLS to avoid collinearity. XGBoost used as primary model for better cross-era generalisation (Test R²: 0.371 vs RF 0.237).
era_code
Author classification. Original thesis contribution. Maps seasons to software eras based on known F1 technology adoption milestones (1994, 2000, 2006, 2012, 2018).
